This is a timestamp microservice

Developed as an exercise from freecodecamp.com with functionally similar to this: timestamp-ms

User Stories:

  • I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
  • If it does, it returns both the Unix timestamp and the natural language form of that date.
  • If it does not contain a date or Unix timestamp, it returns null for those properties.

Example usage:

  • https://fcc-api-projects-superplan.c9users.io/42      {"unix":42,"natural":"January 1, 1970"}
  • https://fcc-api-projects-superplan.c9users.io/june16,1904      {"unix":-2068416000,"natural":"June 16, 1904"}
  • https://fcc-api-projects-superplan.c9users.io/today      {"unix":1482796800,"natural":"December 27, 2016"} (depending on current date)
